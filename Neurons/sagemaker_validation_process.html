SageMaker Built-in Algorithm Hyperparameter Validation

1
Training Job Submission
You submit training job with hyperparameters:
estimator = sagemaker.XGBoost(...)
estimator.set_hyperparameters({
  'max_depth': 15,
  'learning_rate': 2.0, # Invalid!
  'num_round': 100
})
↓
2
Pre-Training Validation
SageMaker checks against algorithm's validation schema:
Range Check: learning_rate must be 0.0 < x ≤ 1.0
Type Check: max_depth must be integer
Required Check: objective parameter required for XGBoost
↓
3
Validation Result
ValidationException: Hyperparameter learning_rate with value 2.0 is not within the required range [0.0, 1.0]
Training job fails BEFORE starting
↓
4
Fix & Resubmit
You correct the parameter:
estimator.set_hyperparameters({
  'max_depth': 15,
  'learning_rate': 0.1, # Fixed!
  'num_round': 100
})
↓
5
Training Proceeds
✓ All hyperparameters valid
✓ Training starts

Key Point:

Validation happens BEFORE training, not after. SageMaker doesn't analyze your results and suggest parameter improvements - it only prevents invalid configurations from running.

For post-training optimization, use:

• Automatic Model Tuning - Tests parameter combinations
• SageMaker Experiments - Compare different runs
• Manual analysis of training metrics

Example Validation Rules by Algorithm:

XGBoost
• learning_rate: (0, 1]
• max_depth: [1, ∞)
• subsample: (0, 1]
Linear Learner
• learning_rate: (0, ∞)
• l1: [0, ∞)
• epochs: [1, ∞)
